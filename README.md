# RelayBox - 现代化的 Clash 配置转换与链路整理工具

RelayBox 是一款专注于**订阅整理、中转链路规划、智能规则补充**的工具。它能够将机场订阅节点、个性化跳板选择及落地节点，完美整合成一份可直接导入 Clash Verge 或各平台 Clash 客户端的 `YAML` 配置文件。

---

## 🚀 核心功能特性

### 1. 灵活的代理模式
- **中转模式 (Relay)**：支持选择一个或多个跳板节点，通过 `dialer-proxy` 机制将落地节点级联在中转链路上。导出配置自动生成自动选择、故障转移、负载均衡三子组。
- **直连模式 (Direct)**：跳过跳板，直接生成落地节点的直连配置，适合简单场景。
- **订阅整理模式**：不设置特定落地节点，仅针对原订阅进行节点筛选、分组重组与规则清洗。

### 2. 🧩 规则助手 (Rule Assistant)
- **智能解析**：粘贴域名、URL 或关键词，自动推荐 `DOMAIN`、`DOMAIN-SUFFIX`、`DOMAIN-KEYWORD` 候选。
- **冲突检测**：自动分析当前配置，识别规则冲突（重复条目、宽泛/精确匹配重叠）。
- **快捷片段**：内置常用 AI 服务（OpenAI, Claude）、开发社区（GitHub）、工具站点的快速规则包。
- **最近历史**：保留最近 10 条手动添加记录，支持一键复用。

### 3. 📥 多维数据导入
- **增强型订阅抓取**：支持 Base64、纯节点列表、Clash YAML。网页版支持通过本地代理绕过 CORS 限制。
- **[NEW] Clash 配置粘贴**：当网页端无法直接拉取订阅时，可直接将 Clash Verge 中的完整配置内容粘贴进来，工具将自动提取其中的所有有效节点。

### 4. 🧹 智能节点治理
- **信息节点过滤**：内置强大的正则引擎，自动识别并隐藏机场的"剩余流量"、"有效期"、"官网地址"等非代理公告节点。
- **一键清理**：支持从工作台中彻底移除识别出的信息节点，让节点选择器清爽如初。

## 📖 使用教程

1. **获取节点**：
   - 填入订阅地址点击“抓取”。
   - 或者使用“从 Clash Verge 粘贴配置”功能导入节点。
2. **选择模式**：
   - 在顶部切换 **【中转模式】** 或 **【直连模式】**。
3. **规划链路 (中转模式)**：
   - 在节点列表中勾选想要作为跳板的节点（可多选）。
   - 被选中的节点将自动组成“前置跳板组”。
4. **配置落地**：
   - 输入落地节点的别名与参数，或通过链接解析导入。
5. **补充规则**：
   - 使用 **规则助手** 快速搜索并添加自定义规则。
   - 调整策略（如：使 AI 走落地，国内走直连）。
6. **导出配置**：
   - 点击“生成配置”，预览生成的 YAML。
   - 点击“一键复制”或“下载 YAML”即可导入客户端使用。

## 🏗️ 技术架构

### 核心分层
- **UI 层**：基于 Vue 3 + Element Plus 的响应式设计，适配桌面与移动端。
- **Composables (逻辑层)**：
  - `useConfig`: 负责 Clash YAML 的生成逻辑。
  - `useNodes`: 负责节点的筛选、排序与隐藏。
  - `useSubscription`: 负责多渠道数据源的解析与标准化。
- **桥接层**：支持桌面端 (Tauri) 原生网络请求与网页端 (CORS Proxy) 代理模式。

### 质量保障
- **单元测试**：使用 Node.js 原生 Test Runner 覆盖 `buildClashConfig` 等核心纯函数，确保各种边界情况（如悬空跳板、多跳板、订阅模式兼容性）稳定无虞。
