# RelayBox - Clash 配置生成器

RelayBox 现在只做一件事：把机场订阅节点、跳板选择和落地节点整理成一份可直接导入客户端的 `Clash YAML`。

## 当前能力

- 订阅抓取与节点解析
  - 支持 Base64 文本、纯节点列表、Clash YAML 订阅内容
  - 支持本地代理地址转发抓取，用来绕过 CORS
- 跳板与直连两种模式
  - 中转模式：落地节点通过一个或多个跳板节点连接
  - 直连模式：无跳板，只生成落地节点直连配置
- 落地节点配置
  - `SOCKS5` / `HTTP` 支持手动填写
  - `SS`、`VMess`、`VLESS`、`Trojan`、`Hysteria(2)`、`TUIC`、`AnyTLS` 统一通过节点链接解析导入
- Clash 规则
  - 内置默认规则始终启用
  - 支持额外追加自定义规则文本，自定义规则优先级更高
- 配置导出
  - 在线预览 YAML
  - 一键复制
  - 下载 `config.yaml`

## 主流程

1. 填入机场订阅地址，必要时补上本地代理地址。
2. 获取节点后，决定使用 `中转模式` 还是 `直连模式`。
3. 配置落地节点。
4. 补充自定义规则（可选）。
5. 生成并导出 Clash 配置。

## 不再提供的旧能力

这些已经不在产品范围内了：

- `V2Ray` / `V2rayN` 导出
- 外部 `rule-providers`
- 自动刷新订阅
- 节点健康监控
- 模板保存 / 加载
- 配置文件导入
- 规则模板面板

## 本地开发

### 环境

- Node.js 18+
- npm

### 安装依赖

```bash
npm install
```

### 启动开发环境

```bash
npm run dev
```

会同时启动：

- Web 前端：`http://localhost:5173`
- 本地订阅代理：`http://localhost:8787`

### 构建

```bash
npm run build
```

### 回归测试

```bash
npm test
```

当前测试覆盖的关键路径：

- 中转模式单跳板
- 中转模式多跳板
- 直连模式无跳板
- 复杂协议必须通过链接解析
- 自定义规则优先级与去重
- 订阅抓取错误提示分类
