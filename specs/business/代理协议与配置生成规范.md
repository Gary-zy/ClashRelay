# 代理协议与配置生成规范

## 支持协议

当前节点解析入口是 `src/utils/parsers.js` 的 `parseProxyLine(line, index)`。

| 协议 | 链接前缀 | 解析函数 | 备注 |
|------|----------|----------|------|
| VMess | `vmess://` | `parseVmess` | Base64 JSON |
| VLESS | `vless://` | `parseVless` | URLSearchParams |
| Trojan | `trojan://` | `parseTrojan` | URLSearchParams |
| AnyTLS | `anytls://` | `parseAnyTLS` | URLSearchParams |
| Shadowsocks | `ss://` | `parseSS` | SIP002 兼容 |
| ShadowsocksR | `ssr://` | `parseSSR` | Base64 |
| Hysteria | `hysteria://` | `parseHysteria` | URLSearchParams |
| Hysteria2 | `hysteria2://`, `hy2://` | `parseHysteria2` | URLSearchParams |
| TUIC | `tuic://` | `parseTUIC` | URLSearchParams |
| SOCKS5 | `socks5://`, `socks5h://`, `socks://` | `parseLandingNodeUrl` 内处理 | 当前只作为落地节点链接 |
| HTTP | `http://` | `parseLandingNodeUrl` 内处理 | 当前只作为落地节点链接 |

新增协议时必须同步：

- `parseProxyLine`
- 对应 `parseXxx` 函数
- `landingTypeOptions`
- `validateLandingProxyInput` 的必填字段
- `parseLandingNodeUrl` 的提示文案
- `tests/parsers.test.js`
- 本文件

## 端口与必要字段

所有协议解析必须拒绝非法端口：

- 必须是整数。
- 范围必须是 `1-65535`。
- 不能让 `0`、空字符串、非数字、浮点数通过。

节点缺关键字段时必须返回 `null` 或 `{ ok: false }`，不能生成半截 proxy 对象。

## 落地节点输入

落地节点有两类：

| 类型 | 来源 | 说明 |
|------|------|------|
| 手动类型 | `socks5`、`http` | 允许用户用表单字段填写地址、端口、用户名、密码 |
| 链接解析类型 | 其他协议 | 必须通过节点链接解析导入，避免表单字段不完整 |

落地节点别名不能为空，且不能和订阅节点名冲突。

## 配置生成入口

| 函数 | 模式 | 说明 |
|------|------|------|
| `buildClashConfig` | 中转模式、直连模式 | 需要落地节点；中转模式至少选择一个有效跳板 |
| `buildSubscriptionOnlyConfig` | 订阅整理模式 | 不需要落地节点；所有订阅节点进入代理出口 |
| `generateYaml` | UI 入口 | 根据 `form.generateMode` 选择生成函数 |

生成 YAML 使用：

```js
yaml.dump(result.config, {
  noRefs: true,
  lineWidth: 120,
  quotingType: '"',
});
```

不要手写大段 YAML 字符串。

## 中转模式配置规则

中转模式下：

1. 订阅节点先经过 `completeNodes` 补全。
2. 用户选择的 `dialerProxyGroup` 会按真实节点名过滤，避免悬空引用。
3. 如果选择覆盖完整来源组，会压缩成来源策略组，减少前置组长度。
4. 单个前置策略时，落地节点的 `dialer-proxy` 直接引用该策略。
5. 多个前置策略时，创建 `🔀 前置跳板组`，落地节点的 `dialer-proxy` 引用这个组。
6. 落地节点进入 `🎯 <落地节点名>` 策略组。
7. `🌐 代理出口` 负责汇总落地组、自动选择、故障转移、负载均衡、来源组、原始节点和 `DIRECT`。

固定组名：

- `🌐 代理出口`
- `🎯 <落地节点名>`
- `🔀 前置跳板组`
- `♻️ 自动选择`
- `🛡️ 故障转移`
- `⚖️ 负载均衡`

改这些名字会影响规则替换、健康检查、测试和用户认知，必须谨慎。

## 直连模式配置规则

直连模式下：

- 不要求选择跳板。
- 落地节点不注入 `dialer-proxy`。
- 仍生成 `🎯 <落地节点名>` 和 `🌐 代理出口`。
- `🌐 代理出口` 只包含落地组和 `DIRECT`。

## 订阅整理模式配置规则

订阅整理模式下：

- 不需要落地节点。
- 所有订阅节点进入导出配置。
- 生成来源组、自动选择、故障转移、负载均衡。
- `{{LANDING}}` 不应保留在最终规则中，必须归并到 `🌐 代理出口` 或当前代理策略。

## 默认规则

默认规则在 `src/config/defaultConfig.js`。

| 数组 | 使用场景 |
|------|----------|
| `defaultRules` | 中转模式和直连模式 |
| `subscriptionDefaultRules` | 订阅整理模式 |
| `fakeIpFilter` | DNS fake-ip 过滤 |

规则占位符：

- `{{LANDING}}`：落地专线。
- `{{PROXY}}`：通用代理出口。

自定义规则优先于内置规则，最终通过 `dedupeRules` 去重。

## 导出前校验

所有生成配置都必须经过 `validateClashConfig(config)`。

校验范围：

- 代理名重复。
- 代理组名重复。
- 代理名和代理组名冲突。
- `dialer-proxy` 引用不存在。
- 代理组引用不存在策略。
- 规则引用不存在策略。

新增配置生成路径时，也必须调用同等级校验。

## Shadowrocket 输出

Shadowrocket 输出目标由 `OUTPUT_TARGETS` 和 `form.outputTarget` 控制。调整兼容逻辑时必须同时验证：

- Clash 默认输出不被破坏。
- Shadowrocket YAML 可导入。
- 中转链路仍符合“客户端 -> 前置跳板 -> 落地节点 -> 目标网站”。

## 文档同步要求

改动以下内容时必须更新本文件：

- 新增协议或改协议解析字段。
- 改策略组命名、结构或排序。
- 改默认规则或占位符语义。
- 改配置校验策略。
- 改 Shadowrocket 兼容输出。
