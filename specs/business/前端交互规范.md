# 前端交互规范

## 视觉与语气

当前 RelayBox 前端是桌面工作台风格，带水墨装饰和中文操作文案。交互文案可以接地气，但必须准确，尤其是错误原因、端口范围、导入限制和配置风险。

## 工作台结构

主页面由以下区域组成：

1. 顶部品牌与模式状态。
2. 工作模式切换：落地节点模式、直连模式、订阅整理模式。
3. 工作区 Tabs：获取订阅、节点工作台、落地配置、规则与导出。
4. 输出预览、配置健康检查、mihomo 校验与测速。

移动或重排这些区域时，必须保证主路径仍清楚：

```text
导入节点 -> 筛选/选择节点 -> 配置落地或订阅整理 -> 规则补充 -> 生成/校验/导出
```

## 模式交互

| 模式 | 节点区 | 落地区 | 导出 |
|------|--------|--------|------|
| 落地节点模式 | 可选择前置跳板 | 必须配置落地节点 | 生成链式代理配置 |
| 直连模式 | 节点仅参考和测速 | 必须配置落地节点 | 生成落地直连配置 |
| 订阅整理模式 | 所有节点默认参与导出 | 展示说明，不要求落地 | 生成纯订阅整理配置 |

切模式时要避免保留无效 UI 状态。例如订阅整理模式不应该继续要求落地节点。

## 导入交互

当前支持四条导入路径：

1. 订阅 URL 抓取。
2. Clash Verge 完整配置粘贴，只提取 `proxies`。
3. 追加融合 Clash 配置，按来源命名并默认选为跳板。
4. 手动粘贴订阅内容，支持 Base64、YAML、逐行节点链接。

Web 端订阅 URL 抓取应通过 `proxyUrl /fetch` 绕过浏览器 CORS；公网生产部署默认把 `proxyUrl` 设为当前站点 origin，避免浏览器直接请求机场订阅地址。

订阅 URL 输入框不提供历史下拉，也不维护单独的订阅地址缓存；用户清空输入后，不应再从旧订阅地址列表中回填。输入框必须关闭浏览器自动补全，订阅地址也不进入主配置持久化。

主动导入节点成功后应切到节点工作区，包括订阅 URL 抓取、手动粘贴订阅内容、Clash 配置提取和融合导入确认。后台状态变化只负责更新步骤状态，不应直接抢走用户当前查看的工作区。

导入成功后：

- 新节点要经过重名处理。
- 节点默认 `latency = -1`。
- 原选择中不存在的新节点名要被裁剪，避免悬空选择。
- 信息节点默认隐藏，但允许用户查看或清理。

## 节点工作台

节点区必须支持：

- 搜索节点名、服务器、协议类型。
- 按默认、延迟、名称、类型排序。
- 来源分组优先，其次区域分组。
- 全选当前显示、反选当前显示、清空选择。
- 信息节点隐藏和清理。
- Web 端通过 `proxyUrl /ping` 测速，桌面端通过 Tauri 原生测速。
- 节点长列表应优先使用页面外层自然滚动，不要把主节点表格固定成小框内滚动。

测速并发限制当前为 10，改动时必须说明理由并补测试或手动验证说明。

## 规则交互

规则助手必须保证：

- 候选规则能根据输入自动推断。
- 能展示重复或宽泛冲突。
- 最近使用记录最多保留 10 条。
- 策略选项随当前模式变化。
- 自定义规则优先于内置规则。

规则文本中允许注释和空行，最终解析时忽略。

## 状态提示

所有用户可感知失败都必须给明确提示：

- 空输入。
- URL 格式不合法。
- 订阅源 HTTP 错误。
- CORS 或本地代理不可用。
- YAML 非法。
- 没有 `proxies` 段。
- 节点解析为空。
- 端口非法。
- 选择的跳板已不存在。
- 规则引用不存在策略。

错误提示不要只写“失败了”，要告诉用户下一步该检查啥。

## 按钮与禁用态

会修改状态的按钮必须有明确禁用条件：

- 没有粘贴内容时不能导入。
- 融合导入未填写来源名称时不能预览。
- 未生成 YAML 时不能校验或导出。
- 正在请求或测速时避免重复触发。

## 响应式要求

改 UI 后至少检查：

- 桌面宽屏。
- 窄屏或移动宽度。
- 文案不会挤出按钮或遮挡内容。
- Tabs、折叠面板、状态 Badge 不互相覆盖。

## 文档同步要求

改动以下内容时必须更新本文件：

- 改工作台步骤或模式文案。
- 新增导入路径。
- 改节点选择、测速、筛选逻辑。
- 改规则助手行为。
- 改主要错误提示策略。
