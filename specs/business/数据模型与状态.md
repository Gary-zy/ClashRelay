# 数据模型与状态

## 表单状态 `form`

`src/App.vue` 中的 `formDefaults` 是当前主表单的源头。

| 字段 | 类型 | 说明 |
|------|------|------|
| `subscriptionUrl` | string | 机场订阅地址 |
| `proxyUrl` | string | Web 端订阅代理地址；开发环境默认 `http://localhost:8787`，生产 Web 默认当前站点 origin |
| `landingNodeUrl` | string | 落地节点链接输入 |
| `landingNodeType` | string | 落地节点协议类型 |
| `socksServer` | string | 手动 SOCKS5/HTTP 落地地址 |
| `socksPort` | string | 手动 SOCKS5/HTTP 落地端口 |
| `socksUser` | string | 手动落地用户名 |
| `socksPass` | string | 手动落地密码 |
| `socksAlias` | string | 落地节点别名，默认 `落地节点` |
| `dialerProxyGroup` | string[] | 被选为前置跳板的节点名 |
| `dialerProxyType` | string | 前置跳板组类型：`url-test`、`select`、`fallback` |
| `urlTestInterval` | number | 前置跳板测速间隔 |
| `urlTestTolerance` | number | url-test 容差 |
| `urlTestLazy` | boolean | 是否启用 lazy 行为 |
| `customRulesText` | string | 用户自定义规则文本 |
| `isDirect` | boolean | 是否直连模式 |
| `generateMode` | string | `relay` 或 `subscription` |
| `outputTarget` | string | 输出目标客户端，默认 Clash |

新增、删除或改名字段时，必须同步：

- `formDefaults`
- `persistedKeys`
- `desktopPersistedKeys`，如果桌面端也要保存
- `assignFormSafely`
- 相关测试
- 本文件

## 节点对象

节点对象遵循 Clash proxy 对象结构，常见字段：

| 字段 | 说明 |
|------|------|
| `name` | 节点名，必须唯一 |
| `type` | 协议类型，如 `vmess`、`vless`、`ss`、`trojan` |
| `server` | 服务器地址 |
| `port` | 端口，必须是 1-65535 整数 |
| `uuid` | VMess/VLESS/TUIC 等协议 UUID |
| `password` | SS/Trojan/Hysteria2/TUIC 等密码 |
| `cipher` | SS/VMess 等加密方式 |
| `network` | `ws`、`grpc`、`tcp`、`h2`、`kcp`、`quic` 等 |
| `dialer-proxy` | 导出时用于链式代理的前置策略 |
| `latency` | UI 测速状态，不属于 Clash 必要字段 |
| `sourceName` | 融合导入来源名称 |
| `sourcePrefix` | 融合导入节点名前缀 |

导出前必须使用 `cleanProxyForExport` 清理 UI 临时字段。

## 订阅导入状态

`useSubscription` 维护：

- `isFetching`：订阅抓取中。
- `parseSubscription(text)`：支持 Base64、Clash YAML、逐行节点链接。
- `parseClashConfigNodes(text)`：只提取 `proxies`，不继承原策略组和规则。
- `previewMergedClashConfigText` / `commitMergedClashConfigPreview`：融合导入两段式流程。

## 持久化

| Key | 存储位置 | 内容 |
|-----|----------|------|
| `clashrelay_config` | localStorage | 主表单状态 |
| `relaybox_rule_helper_recent` | localStorage | 规则助手最近条目 |
| `relaybox_mihomo_token` | localStorage | mihomo API token |
| `relaybox_mihomo_benchmark_history` | localStorage | mihomo 测速历史 |

`subscriptionUrl` 不进入 `clashrelay_config`，桌面端状态恢复也会忽略旧版本保存过的订阅地址，避免订阅入口在刷新或重启后自动回填。

生产 Web 环境会清理残留的 localhost `proxyUrl`，并在为空时恢复为 `window.location.origin`，避免部署后浏览器直连订阅源触发 CORS，也避免默认指向用户本机不可用服务。

## 桌面端状态

桌面端状态 payload：

```js
{
  version: DESKTOP_STATE_VERSION,
  form: { ...desktopPersistedKeys },
  lastSavedPath: ""
}
```

桌面端不持久化 `proxyUrl`，因为桌面壳有原生订阅抓取与测速能力。

## 生成产物状态

| 状态 | 说明 |
|------|------|
| `yamlText` | 当前生成的 YAML 文本 |
| `previousYaml` | 上一次 YAML，用于差异或状态提示 |
| `configHealthReport` | 从 `yamlText` 解析得到的健康报告 |
| `benchmarkPolicies` | 从 `yamlText` 提取的 mihomo 可测速策略 |

## 文档同步要求

改动以下内容时必须更新本文件：

- 新增表单字段。
- 改 localStorage key。
- 改节点对象字段含义。
- 改导入流程或融合导入流程。
- 改桌面端保存结构。
