# 运维手册

## 环境要求

- Node.js >= 20。
- npm。
- Tauri 构建需要 Rust 与系统平台依赖。
- mihomo 校验需要可用 mihomo 二进制，或通过脚本自动选择资源。

## 安装

```bash
npm install
```

## 开发启动

同时启动 Vite 和本地代理：

```bash
npm run dev
```

默认地址：

- 前端：`http://localhost:5173`
- 本地代理：`http://localhost:8787`

只启动 Web 前端：

```bash
npm run dev:web
```

## 构建

Web 构建：

```bash
npm run build
```

桌面模式构建前端：

```bash
npm run build:desktop
```

Tauri 开发：

```bash
npm run tauri:dev
```

Tauri 构建：

```bash
npm run tauri:build
```

Docker 构建：

```bash
npm run docker:build
```

Docker 镜像内会先构建 Web 前端，再由 `proxy-server.js` 托管 `dist`。生产构建的前端资源使用 `/ClashRelay/` 前缀，代理服务必须同时支持根路径访问和 `/ClashRelay/` 静态资源路径，避免 Docker 公网直连部署时资源请求回退成 `index.html`。

## 测试与校验

单元测试：

```bash
npm test
```

构建验证：

```bash
npm run build
```

mihomo 配置合法性验证：

```bash
npm run verify:clash
```

全量验证：

```bash
npm run verify:all
```

## 环境变量

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `PORT` | `8787` | 本地代理服务端口 |
| `STATIC_DIR` | `dist` | 代理服务静态文件目录 |
| `MIHOMO_BIN` | 空 | 指定 mihomo 二进制路径 |
| `RELAYBOX_TOKEN` | 空 | `/mihomo/*` 接口鉴权 token，前端 mihomo 面板需填写相同值 |
| `TAURI_ENV_PLATFORM` | 空 | Tauri 平台标识 |

## 本地代理服务端点

| 端点 | 方法 | 说明 |
|------|------|------|
| `/fetch?url=` | GET | 抓取订阅内容，带 SSRF 防护 |
| `/ping` | POST | TCP 连接测速 |
| `/mihomo/*` | 多种 | mihomo 校验、启动、测速相关 |
| 静态文件 | GET/HEAD | 服务 `dist` 或 `STATIC_DIR`，兼容 `/ClashRelay/` 前缀资源路径 |

## 安全要求

新增或修改代理服务逻辑时必须保留：

- CORS 白名单和同源判断。
- 请求体 1MB 限制。
- SSRF 私有/保留地址拦截。
- DNS 解析后地址检查。
- 重定向最多 3 次。
- 静态文件路径限制在 `STATIC_DIR` 内。
- 错误响应不要泄露敏感本地路径或 token。

## 常见问题

### Web 端抓订阅失败

检查：

1. 订阅 URL 是否是 http/https。
2. 订阅源是否返回 401、403、404 或 5xx。
3. 如果直接跨域失败，是否启动了 `npm run dev` 中的代理服务。
4. `form.proxyUrl` 是否指向 `http://localhost:8787`。

### 桌面端抓订阅失败

检查：

1. Tauri 宿主网络权限。
2. 订阅源是否拒绝连接。
3. 桌面端错误分类是否落到 `network` 或 HTTP 状态。

### 生成配置不能导入 Clash

检查：

1. 页面配置健康报告。
2. `validateClashConfig` 错误。
3. `npm run verify:clash` 输出。
4. 是否存在重复代理名、悬空策略组、非法端口。

### mihomo 校验失败

检查：

1. `MIHOMO_BIN` 是否可执行。
2. mihomo 端口或 token 是否冲突。
3. 生成 YAML 是否已通过本地配置校验。
