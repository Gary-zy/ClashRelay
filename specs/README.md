# RelayBox Specs 规范库

这个目录是 RelayBox 后续开发的事实标准。AI 或人工改代码前，必须先读这里；改完代码后，只要行为、结构、数据、配置输出或运维方式变了，也必须同步更新这里。

## 目录结构

```text
specs/
  analysis/
    当前代码结构盘点.md
  business/
    项目总览.md
    架构设计.md
    模块职责.md
    数据模型与状态.md
    代理协议与配置生成规范.md
    前端交互规范.md
    运维手册.md
    AI开发守则.md
```

## 阅读顺序

1. 先读 `business/AI开发守则.md`，明确本次改动需要同步哪些 specs。
2. 再读和任务相关的业务规范。
3. 如果要做重构或拆模块，读 `business/架构设计.md` 和 `business/模块职责.md`。
4. 如果发现代码和 specs 不一致，先核实当前代码真实行为，再更新文档或修代码。

## 维护原则

- specs 写当前真实状态，不写已经移除或想象中的功能。
- specs 不替代 README，README 面向用户，specs 面向开发和 AI 协作。
- 每次功能改动结束前，都要回答一句：本次是否需要更新 specs？需要就更新，不需要就说明原因。
- specs 允许先粗后细，但不允许留 `TODO`、`TBD`、空章节。
