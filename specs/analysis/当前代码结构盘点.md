# 当前代码结构盘点

更新时间：2026-04-30

## 总体判断

RelayBox 当前是一个 Vue 3 + Element Plus 前端、Node.js 本地代理服务、Tauri 桌面壳混合项目。核心业务逻辑已经从早期单文件形态拆出一部分到 `src/composables/` 和 `src/utils/`，但 `src/App.vue` 仍承担较多页面状态、工作台流程和事件处理。

## 根目录

| 路径 | 当前职责 |
|------|----------|
| `package.json` | npm 脚本、依赖版本、Node 测试入口 |
| `vite.config.js` | Vite 构建配置 |
| `proxy-server.js` | 本地代理、静态资源服务、订阅抓取、节点 ping、mihomo 接口 |
| `Dockerfile` | 容器构建入口 |
| `README.md` | 面向用户的功能说明和使用说明 |
| `AGENTS.md` | 面向 AI 的开发入口和强制规范入口 |
| `specs/` | 后续开发必须同步维护的规范库 |
| `tests/` | Node.js 原生测试用例 |
| `scripts/` | mihomo 资源选择与校验脚本 |
| `src-tauri/` | Tauri 桌面端配置和 Rust 宿主代码 |

## 前端源码

| 路径 | 当前职责 |
|------|----------|
| `src/main.js` | Vue 应用初始化，注册 Element Plus |
| `src/App.vue` | 主工作台、模式切换、表单状态、导入导出、mihomo 操作、页面交互 |
| `src/styles.css` | 全局主题、布局、组件覆盖样式 |
| `src/components/` | 独立 UI 组件，如引导、选择托盘、配置健康面板、加载态 |
| `src/components/decorations/` | 水墨风装饰组件 |
| `src/composables/useConfig.js` | 落地节点解析、输入校验、Clash/订阅配置生成 |
| `src/composables/useSubscription.js` | 订阅抓取、Clash YAML 节点提取、融合导入 |
| `src/composables/useNodes.js` | 节点搜索、分组、筛选、选择、测速 |
| `src/composables/useNodeParams.js` | 节点默认参数补全 |
| `src/config/defaultConfig.js` | 默认规则、订阅整理默认规则、DNS fake-ip-filter |
| `src/utils/parsers.js` | 节点链接解析和规则文本解析 |
| `src/utils/clashConfigValidator.js` | Clash 配置引用关系校验 |
| `src/utils/proxySanitizer.js` | 导入/导出节点清洗和重名处理 |
| `src/utils/nodeSources.js` | 来源标记、来源分组、完整来源组压缩 |
| `src/utils/ruleAssistant.js` | 规则助手候选生成和冲突分析 |
| `src/utils/mihomoClient.js` | mihomo API 客户端和错误文案 |
| `src/utils/mihomoPolicies.js` | 从配置中提取可测速策略 |
| `src/utils/configHealth.js` | 生成配置健康报告 |
| `src/utils/desktop.js` | Tauri 桌面端 API 适配 |

## 服务端源码

| 路径 | 当前职责 |
|------|----------|
| `proxy-server.js` | HTTP 服务主入口、安全校验、路由分发 |
| `src/server/mihomoBenchmark.js` | mihomo 启动、健康检查、策略测速请求归一化 |

## 测试现状

项目使用 `node --test`。当前测试覆盖解析器、配置生成、代理服务、订阅导入、mihomo 相关逻辑、规则助手、配置健康检查等。

关键命令：

```bash
npm test
npm run build
npm run verify:clash
npm run verify:all
```

## 当前需要留意的结构债

- `src/App.vue` 仍然偏大，新增复杂业务时优先抽到 composable 或 utils。
- 根级历史文档容易滞后，后续以 `specs/` 为开发事实标准。
- Web 和 Tauri 双运行环境有差异，涉及网络、文件保存、测速时必须同时考虑。
- 代理服务器既做静态资源又做 API，新增路由时要特别注意安全边界。
