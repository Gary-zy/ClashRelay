# RelayBox 功能说明

## 产品定位

RelayBox 是一个 `Clash only` 的配置生成工具，目标是把最常用的主路径压缩成四步：

1. 获取订阅节点
2. 选择中转或直连
3. 配置落地节点
4. 生成 Clash YAML

## 核心功能

### 1. 订阅获取

- 支持抓取常见 Clash / Base64 订阅内容
- 支持本地代理服务转发订阅请求
- 自动保存最近使用的订阅历史
- 报错信息按场景区分：
  - 404 / 403 / 5xx
  - 本地代理服务不可达
  - 请求超时
  - 内容为空或解析失败

### 2. 跳板节点选择

- 节点列表支持搜索、分组、排序
- 支持一键测速
- 中转模式支持：
  - 单跳板直接注入 `dialer-proxy`
  - 多跳板生成 `前置跳板组`
  - `url-test` / `select` / `fallback`
- 直连模式下：
  - 不要求选择跳板
  - 跳板区只保留只读参考，不参与生成

### 3. 落地节点配置

- `SOCKS5` / `HTTP` 支持手动填写地址、端口、认证信息
- 复杂协议统一通过链接解析导入：
  - `SS`
  - `SSR`
  - `VMess`
  - `VLESS`
  - `Trojan`
  - `AnyTLS`
  - `Hysteria`
  - `Hysteria2`
  - `TUIC`
- 生成前会校验关键字段和端口合法性，避免产出半残配置

### 4. Clash 配置生成

- 默认规则始终启用
- 自定义规则文本会排在默认规则前面
- 最终规则会做去重
- 保留固定 DNS 默认值，不再暴露一堆 UI 开关
- 支持在线预览、复制、下载

## 质量保障

- `npm run build`：前端构建检查
- `npm test`：Node 内置测试，覆盖主流程关键回归场景

## 明确移除的旧功能

下面这些已经不再是当前产品的一部分：

- `V2Ray` / `V2rayN` 导出
- 外部规则源和规则模板面板
- 自动刷新订阅
- 节点健康监控
- 配置导入
- 模板存取
